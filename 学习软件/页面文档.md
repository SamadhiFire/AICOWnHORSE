## 小程序需求文档

### **1. 页面 1：刷题页面（核心功能）**

#### **功能目标**：
用户在此页面粘贴学习资料，选择题目类型、难度和生成题目数量，点击按钮生成题目。

#### **UI 设计**：
- **底部导航**：按钮显示 **“刷题”**，作为当前页面的导航。
- **顶部**：页面标题 **“我爱刷题”**，突出显示功能。
- **资料输入框**：
  - 允许用户粘贴学习资料，推荐至少 200 字。
- **模式选择**：
  - **模式 A**：严格基于用户提供的资料生成题目。
  - **模式 B**：允许加入外部常识生成题目。
- **题目类型选择**：
  - 单选题、填空题等。
- **题目难度选择**：
  - 简单、中等、困难。
- **题目数量选择**：
  - 允许用户选择题目数量（5题、10题、15题或自定义）。
- **生成按钮**：
  - 点击按钮后，调用 DeepSeek API 生成题目并显示。

#### **调用大模型时机**：
- **生成题目**时，根据用户输入的 **资料**，选择模式后调用大模型（DeepSeek API）生成题目。
- **模式 A**：基于资料生成题目，不允许加入外部常识。
- **模式 B**：除了资料内容外，允许补充常识生成题目。

---

### **2. 页面 2：题集页面（查看已生成的题目）**

#### **功能目标**：
用户查看自己生成的题目，包含题目的汇总和错题展示。

#### **UI 设计**：
- **底部导航**：按钮显示 **“题集”**，作为当前页面的导航。
- **题目汇总**：
  - 显示用户生成的所有题目，包括选择题、填空题等。
  - 每个题目展示 **题目类型**、**难度** 和 **答对/错** 状态。
  - 允许用户 **查看**、**编辑** 或 **删除** 题目。
- **错题本**：
  - 专门展示用户答错的题目，帮助复习。
  - 用户可以点击错题查看详细信息。

#### **功能实现**：
- **题目管理**：
  - 每个题目展示一个 **查看**、**编辑** 和 **删除** 按钮。
  - 点击查看时，展示题目的详细内容（题目、选项、解析等）。
  - 点击删除时，从数据库中删除该题目。

#### **调用大模型时机**：
- **错题本**：当用户点击查看某个错题时，若需要重新生成该题的解析或提示，可以调用大模型提供进一步的帮助（如补充解析、解释）。

### **3. 页面 3：个人信息页面（API Key 配置）**

#### **功能目标**：
用户在此页面查看和配置自己的 **DeepSeek API Key**。

#### **UI 设计**：
- **底部导航**：按钮显示 **“我的”**，作为当前页面的导航。
- **API Key 配置**：
  - 显示当前的 **DeepSeek API Key**，如果为空，显示默认的 `deepseek`。
  - 允许用户修改 API Key，点击保存按钮保存更新后的 API Key 到云数据库。
  
#### **功能实现**：
- **修改 API Key**：用户修改 API Key 后，点击保存按钮更新数据库中的 **API Key**。
- **默认值**：初始显示的 API Key 为 `deepseek`，用户可以修改。

#### **调用大模型时机**：
- **API Key 配置**：如果用户修改了 API Key，前端需要更新 API Key 设置，后续调用 DeepSeek 大模型时，使用更新后的 API Key。

---

### **4. 底部导航设计**

**底部导航**将在所有页面上保持一致，确保用户能够方便地切换页面。

- **刷题页面**：显示 **“刷题”** 按钮。
- **题集页面**：显示 **“题集”** 按钮。
- **个人信息页面**：显示 **“我的”** 按钮。
