# 给后端的项目背景

## 项目是什么

这是一个 `uni-app + Vue 3 + TypeScript` 的 AI 刷题小程序前端项目。
当前真实业务工程目录是 `学习软件/`，不是模板目录，也不是旧的云函数方案。

## 当前业务主链路

1. 用户在首页输入学习材料，选择题型、难度、模式、反馈方式、目标题量。
2. 前端优先请求后端生成题目。
3. 题目生成后，前端进入练习页答题、看解析、记录练习次数和错题状态。
4. 用户在题集页查看全部题、错题、已掌握题，并按标签筛选。
5. 用户在“我的”页面维护昵称、头像、LLM 配置和个人标签。

## 当前技术事实

- 架构已经从“前端直连模型/旧云端同步”切换为“前端 + 独立 REST 后端”。
- 统一响应结构是 `{ code, message, data }`。
- 除 `POST /auth/guest` 外，其余接口都需要 `Authorization: Bearer <token>`。
- 时间字段统一使用毫秒时间戳。
- 字段命名统一使用 `camelCase`。

- 前端也支持从本地存储或环境变量覆盖 Base URL，但现在代码默认值就是上面这条。

## 后端需要承接的能力

`认证与用户`
游客登录、获取当前用户、更新昵称头像、上传头像。

`LLM 配置`
保存当前用户的 provider、apiKey、baseUrl、model，以及 provider 列表和 key 校验结果。

`用户标签`
保存标签、根据目标生成标签建议、对历史题目重打标签。

`题库`
整包同步、分页查询、批量删除、提交作答结果、更新掌握状态、批量更新标签。

`当前练习会话`
保存、读取、清空当前正在做的题目会话。

`题目生成`
生成整套题目，必要时兼容旧的 `generation-jobs` 批次接口。

## 页面和后端能力的对应关系

`首页`
依赖 LLM 配置、用户标签、题目生成。

`练习页`
依赖当前练习会话、题目作答结果、生成任务状态、兼容补批接口。

`题集页`
依赖题库查询、删除、掌握状态更新、标签更新、标签列表。

`我的`
依赖用户资料、头像上传、LLM 配置、provider 列表、标签 AI 生成、历史题目重分类。

## 生成链路的当前真实要求

前端现在默认不是“先生成一批，再慢慢补”。
前端默认是：

- `initialBatchCount = targetCount`
- `POST /questions/generate` 作为主入口
- 希望一次性拿到完整 `session.questions`
- 希望 `generationJob` 同时返回，并且已经是 `completed`，或者至少 `loadedCount >= targetCount`

旧的这些接口现在只是兼容保留：

- `GET /generation-jobs/{jobId}`
- `POST /generation-jobs/{jobId}/batches/{batchIndex}`
- `POST /generation-jobs/{jobId}/cancel`

如果后端已经支持一次性生成整套题，前端正常流程下就不应该再依赖补批。

## 这次跟后端对接的核心目标

不是先讨论所有历史方案，而是先把“前端当前真实依赖什么接口、字段必须怎么返回、哪些接口能先跑通主流程”对齐。
