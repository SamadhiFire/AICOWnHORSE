# 给后端的简短提示词

下面这些都是可以直接复制给后端的短消息。

## 版本 1：第一次发消息时用

```md
我们前端现在准备开始和后端联调了。
项目已经切到“前端 + 独立 REST 后端”架构，不再走旧云函数方案。

我先发你三份最关键的材料：
1. 项目背景
2. 前端当前真实依赖的接口清单
3. 详细数据契约

麻烦你先帮我确认：
1. 哪些接口已经可用
2. 哪些还没上线
3. 每个接口最终请求/响应字段
4. `/questions/generate` 能不能一次性返回整套题目、session、generationJob

我们先把最小主流程跑通，再补非核心接口。
```

## 版本 2：如果你只想发最短版

```md
前端要开始联调了。
请先按我发你的接口清单和字段契约，确认哪些接口已上线、哪些未上线，尤其是 `/questions/generate` 是否已支持一次性返回完整 `session + generationJob + questions`。
```

## 版本 3：专门催出题链路时用

```md
前端现在默认一次性生成整套题，不再以分批补题为主。
请优先把 `/questions/generate` 做成主入口，一次返回完整 `session.questions`，并把 `generationJob.status` 置为 `completed`，这样前端主流程才能稳定联调。
```

## 版本 4：让后端按统一格式回你

```md
麻烦你按这个格式回复我每个接口：
接口名 / 当前状态 / 实际请求体 / 实际响应体 / 与前端文档差异 / 预计联调时间
这样我们能最快把协议对齐。
```
