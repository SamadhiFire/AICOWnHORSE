# 前后端联调接口总文档

这份文档用于直接发给前端联调，所有接口统一以当前后端骨架实现为准。

## 1. 基础信息

- Base URL：`http://localhost:3000/api/v1`
- 线上联调时只需要替换域名，路径保持不变
- 除 `GET /health`、`POST /auth/guest`、`POST /auth/refresh` 外，其余接口都需要：
  - `Authorization: Bearer <token>`
- 请求体默认使用 `application/json`
- 上传头像使用 `multipart/form-data`
- 时间字段统一为毫秒时间戳
- 字段命名默认 `camelCase`
- 题目和知识点对象中保留以下字段名：
  - `evidence_quote`
  - `keypoint_id`
  - `importance_score`
  - `why_important`

## 2. 统一响应结构

成功：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

失败：

```json
{
  "code": 40001,
  "message": "material is required",
  "data": {}
}
```

## 3. 常用错误码

- `40001` 参数错误
- `40002` 业务校验失败
- `40100` 未登录或 token 失效
- `40300` 无权限
- `40400` 资源不存在
- `40900` 并发冲突或重复提交
- `42900` 频率限制
- `50000` 服务异常
- `50010` 题目生成失败或生成结果校验失败

## 6. 前端联调建议

- 登录后优先缓存 `token` 和 `refreshToken`
- 题目生成主流程只认 `POST /questions/generate`
- 判断生成成功时，请至少校验：
  - `data.session` 存在
  - `data.generationJob` 存在
- 会话恢复优先使用 `GET /practice-session/current`
- 头像上传字段名固定为 `file`
- `DELETE /question-bank` 当前后端支持 body 传 `ids`
- `GET /llm/providers` 返回的是 `{ providers: [...] }`，不是数组直出
- `StoredQuestion` / `PracticeSession` / `GenerationJob` 都按这份文档字段读取，不要自行裁剪
