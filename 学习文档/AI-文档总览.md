# AI-文档总览

整理时间：2026-03-11

## 这份总览是干什么的

这份文档的目标，不是重复抄一遍原文，而是帮你快速看懂 `AI-` 这个仓库里“每份文档到底在讲什么、给谁看、现在还有没有用”。

本次整理范围以“项目自己写的文档和关键配置文件”为主，排除了下面这些内容：

- `node_modules/` 里的第三方依赖说明
- `dist/` 里的构建产物
- 根目录和子项目里的 `package-lock.json` 这类锁文件

## 先看目录：这个仓库现在怎么分

### `学习软件/`

这是当前真正的主项目目录，也就是现在在开发的刷题小程序工程。  
技术栈是 `uni-app + Vue 3 + TypeScript`，页面、接口调用、打包配置基本都在这里。

### `学习文档/`

这是项目说明、接口文档、联调材料的集中存放处。  
如果你想先搞懂项目业务和前后端分工，优先看这里。

### `uni-preset-vue-vite/`

这是一个比较标准的 `uni-app` 官方模板/脚手架参考目录。  
它更像“模板样板”或“对照组”，不是当前业务项目本体。

### `ai牛马刷题/`

这个目录目前是空的，暂时没有有效文档或代码内容。

## 核心文档分别代表什么

### `学习文档/项目信息文档.md`

这是一份“项目现状快照”。

它主要说明：

- 这个项目是做什么的
- 当前主流程是什么
- 现在已经改成了什么技术架构
- 前端有哪些页面
- 核心工具模块分别负责什么
- 已经接入了哪些后端能力

适合谁看：

- 新接手项目的人
- 想快速知道项目全貌的人

当前价值：

- 很高
- 它是最像“项目总说明书”的文档

### `学习文档/当前前端所需后端接口文档.md`

这是一份“前端当前真实依赖的接口清单”。

它不是泛泛而谈，而是直接按代码里实际调用的位置来写，告诉你：

- 哪个接口被哪个前端文件使用
- 请求体和返回体大概长什么样
- 前端现在真正依赖哪些字段
- 生成题目、题库、标签、会话这些能力怎么分

适合谁看：

- 前端开发
- 后端开发
- 联调时确认“代码现在到底在等什么”

当前价值：

- 非常高
- 如果你只想知道“前端现在到底需要后端给什么”，这份最直接

### `学习文档/后端接口文档.md`

这是一份“比较完整的后端接口规格说明”。

它的特点是：

- 写得比“当前前端所需后端接口文档”更全
- 把接口分成 `P0 已实现` 和 `P1 未实现/计划中`
- 有更完整的请求/响应示例
- 还包含健康检查、错误码建议、生成任务、标签 AI 等规划内容

适合谁看：

- 后端设计接口时
- 前后端统一字段格式时

当前价值：

- 高
- 但它有一个明显特点：文档里的“P1 计划中”状态，和当前前端代码需求之间已经有一点时间差

换句话说：

- 它更像“后端总规范”
- 但不一定完全等于“前端此刻真实已接入状态”

### `学习文档/发给后端的接口问题清单.md`

这不是接口说明书，而是一份“联调前的问题单”。

它的作用是把前端现在卡住的点一次性整理给后端，比如：

- 哪些 `P1` 接口到底有没有真上线
- 头像上传到底怎么传
- `refreshToken` 返回了，但刷新接口在哪
- `PATCH` / `DELETE body` 兼容性稳不稳
- 生成题目的返回字段到底是不是完整题目对象

适合谁看：

- 前后端沟通时
- 项目负责人排联调风险时

当前价值：

- 很高
- 它不是“系统说明”，但非常能反映这个项目现在真正的阻塞点

### `学习文档/前端联调对接指南.md`

这是一份“前端怎么接后端”的操作型文档。

它主要讲：

- Base URL 怎么配
- token 怎么拿
- 常见错误码怎么处理
- fetch/axios 示例怎么写
- 重试、超时、CORS、分页这些工程问题怎么统一

适合谁看：

- 具体写接口请求的人
- 想知道前端接入约定的人

当前价值：

- 很高
- 它比“后端接口文档”更偏前端落地

## `学习软件/` 里的文档分别代表什么

### `学习软件/页面文档.md`

这是一份比较早期的“产品需求 + 页面草图说明”。

它描述了：

- 刷题页要做什么
- 题集页要做什么
- 我的页要做什么
- 底部导航怎么分
- 模式 A / 模式 B 的概念
- 当时设想由 DeepSeek 直接参与哪些能力

当前价值：

- 中等
- 它更像早期需求草案，而不是当前实现真相

需要注意的地方：

- 文档里主要是 3 个页面
- 但当前真实工程已经是 4 个页面：`首页 / 练习页 / 题集页 / 我的页`
- 文档里很多描述还是“前端直接调大模型”的思路
- 而当前项目已经明显转向“前端 + 独立 REST 后端”架构

所以这份文档适合用来理解最初产品想法，不适合直接拿来当现状说明。

### `学习软件/模式A.md`

这是一份“模式 A 出题提示词规则”。

它的核心意思是：

- 严格基于用户提供的材料出题
- 不能补外部常识
- 每道题的答案都必须能在材料里找到证据
- 返回结构里要包含题型、题干、选项、答案、解析、证据

当前价值：

- 高
- 它不是给用户看的产品文档，而是给“出题逻辑/提示词设计”看的规则文档

### `学习软件/模式B.md`

这是一份“模式 B 出题提示词规则”。

它和模式 A 的区别是：

- 允许补充常识
- 但常识不能偏离材料核心概念
- 题目仍然要围绕材料本身

当前价值：

- 高
- 它本质上也是提示词设计文档

简单理解：

- `模式A.md` 代表“严格贴材料”
- `模式B.md` 代表“材料为主，允许合理扩展”

### `学习软件/README.uniapp.md`

这是一份“工程迁移和打包说明”。

它主要告诉你：

- 怎么安装依赖
- 怎么构建微信小程序
- 构建产物在哪
- 怎么用微信开发者工具打开
- 调大模型时要注意什么

当前价值：

- 中等
- 它更偏工程启动说明

需要注意的地方：

- 它提到 “API key 先存本地”
- 但当前 `学习文档` 里的接口文档已经把 LLM 配置写成后端接口能力

所以这份 README 有一定参考价值，但明显带有过渡时期痕迹。

## 关键配置文件分别代表什么

这些文件不是传统意义上的“说明文档”，但它们本身就是工程规则说明，所以也值得一起读懂。

### `学习软件/package.json`

这是主项目的“工程说明书”。

它告诉你：

- 这个项目的名字是什么
- 用了哪些依赖
- 怎么启动和打包
- 常用脚本有哪些

从这份文件可以读出：

- 当前项目主打微信小程序构建
- 有 `type-check`
- 有 `build:mp-weixin`

### `学习软件/tsconfig.json`

这是 TypeScript 编译规则。

它说明：

- 项目开启了 `strict`
- 使用 `ES2020`
- 用 `Bundler` 方式解析模块
- 需要检查哪些目录和文件

它的意义不是业务说明，而是告诉你“代码按什么类型规则写”。

### `学习软件/project.config.json`

这是微信开发者工具项目配置。

它的意义是：

- 告诉微信开发者工具小程序构建产物在哪
- 配置压缩、增强编译等打包行为
- 绑定当前微信小程序 `appid`

### `学习软件/project.private.config.json`

这是本地私有配置。

它通常和个人开发环境有关，比如：

- 项目名
- 是否关闭域名检查
- 是否开启热重载

更偏“本机开发体验配置”，不是业务文档。

### `学习软件/src/pages.json`

这是页面路由与全局页面风格配置。

它说明了当前真实页面结构：

- `pages/index/index`
- `pages/practice/index`
- `pages/collection/index`
- `pages/profile/index`

如果你想确认“现在工程到底有哪些页面”，这份比早期页面文档更可信。

### `学习软件/src/manifest.json`

这是应用清单文件。

它主要说明：

- 应用名称
- 应用描述
- 版本号
- 小程序相关配置
- 微信小程序 `appid`

## `uni-preset-vue-vite/` 这部分代表什么

### `uni-preset-vue-vite/package.json`

这是一个标准 `uni-app` 模板工程的依赖和脚本配置。  
它支持很多平台构建，不只是微信小程序。

它的意义更像：

- 官方模板参考
- 脚手架样例
- 对照当前主项目时的参考基线

### `uni-preset-vue-vite/src/pages.json`

这是模板工程的默认路由配置。  
只有一个默认首页，明显不是你现在业务项目的真实页面结构。

### `uni-preset-vue-vite/src/manifest.json`

这是模板工程的默认清单配置。  
里面大多是空白或默认值，主要用来说明“uni-app 模板原生长什么样”。

结论就是：

- `uni-preset-vue-vite/` 更像参考模板
- `学习软件/` 才是当前真正业务工程

## 我对当前仓库文档状态的总判断

### 1. 主工程已经很明确

当前真正应该围绕的目录是 `学习软件/`，不是 `uni-preset-vue-vite/`，更不是空的 `ai牛马刷题/`。

### 2. 架构已经发生过一次明显迁移

从文档整体来看，这个项目经历过一轮从：

- 前端直接调大模型 / 云端同步思路

转向：

- 前端 + 独立 REST 后端

所以现在有些文档会同时保留“旧思路痕迹”和“新接口规范”。

### 3. 有几份文档明显偏旧

相对偏旧或偏草案性质的主要是：

- `学习软件/页面文档.md`
- `学习软件/README.uniapp.md`
- `学习文档/后端接口文档.md` 里部分 `P1` 状态说明

不是说它们没用，而是要带着“历史参考”心态去看。

### 4. 目前最该当真看的文档

如果你只想抓“现在项目真实情况”，优先信这三份：

- `学习文档/项目信息文档.md`
- `学习文档/当前前端所需后端接口文档.md`
- `学习文档/前端联调对接指南.md`

## 一句话总结

这个仓库现在的文档体系，本质上分成两部分：

- `学习文档/` 负责讲“项目现状、接口规范、联调问题”
- `学习软件/` 负责承载“真实工程”和“早期页面/模式规则”

其中最核心的事实是：

- 真正的业务工程在 `学习软件/`
- 真正的现状说明主要在 `学习文档/`
- `uni-preset-vue-vite/` 只是模板参考
