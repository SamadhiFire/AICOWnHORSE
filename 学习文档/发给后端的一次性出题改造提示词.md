# 发给后端的一次性出题改造提示词

请按下面要求改造出题链路，目标是把“分批生成”切换为“后端一次性生成整套题目，前端只负责展示和状态承接”。

## 目标

1. 以后端作为唯一生成源。
2. `POST /questions/generate` 成为唯一主生成入口。
3. 前端已经切换为默认 `initialBatchCount = targetCount`，也就是一次性请求整套题目。
4. 前端仍保留了旧批次接口兼容代码，但期望正常情况下不再触发 `/generation-jobs/{jobId}/batches/{batchIndex}`。

## 你需要实现的行为

### 1. `/questions/generate` 一次性返回整套题目

请求示例：

```json
{
  "material": "学习材料全文",
  "type": "single",
  "difficulty": "medium",
  "mode": "modeA",
  "feedbackMode": "instant",
  "targetCount": 20,
  "initialBatchCount": 20,
  "userTags": ["网络基础", "数据分析"],
  "requestNonce": 1760000000000
}
```

请确保：

- 直接生成 `targetCount` 道题
- `session.questions.length >= targetCount`
- `savedCount >= targetCount`
- `generationJob.loadedCount >= targetCount`
- `generationJob.status = "completed"`
- `batchState.batch2.requestedCount = 0`
- `batchState.batch3.requestedCount = 0`

推荐返回示例：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "savedCount": 20,
    "keypoints": [],
    "session": {
      "id": "session_1760000000000",
      "createdAt": 1760000000000,
      "mode": "modeA",
      "feedbackMode": "instant",
      "questions": []
    },
    "generationJob": {
      "jobId": "job_1760000000000",
      "sessionId": "session_1760000000000",
      "targetCount": 20,
      "loadedCount": 20,
      "status": "completed",
      "batchState": {
        "batch1": { "index": 1, "requestedCount": 20, "loadedCount": 20, "status": "done", "attempts": 1, "error": "" },
        "batch2": { "index": 2, "requestedCount": 0, "loadedCount": 0, "status": "done", "attempts": 0, "error": "" },
        "batch3": { "index": 3, "requestedCount": 0, "loadedCount": 0, "status": "done", "attempts": 0, "error": "" }
      }
    }
  }
}
```

### 2. question schema 必须固定

每道题统一为：

```json
{
  "id": "string",
  "type": "single | multi",
  "stem": "string",
  "tag": "string",
  "options": [
    { "key": "A", "text": "string" },
    { "key": "B", "text": "string" },
    { "key": "C", "text": "string" },
    { "key": "D", "text": "string" }
  ],
  "answer": "A | A,C",
  "explanation": "string",
  "evidence_quote": "string",
  "keypoint_id": "kp_0",
  "difficulty": "easy | medium | hard"
}
```

知识点统一为：

```json
{
  "id": "kp_0",
  "title": "string",
  "importance_score": 1,
  "evidence_quote": "string"
}
```

## Mode A / Mode B 规则

### Mode A

- 严格基于材料
- 正确答案必须能被材料直接支持
- 不允许材料外知识成为正确项依据

### Mode B

- 允许补充与材料紧密相关的常识
- 但材料仍然是题目的中心
- 外部知识不能压过材料本身

## 题目约束

- `single` 必须是 4 选 1
- `multi` 必须是 4 选 2 或至少 2 个正确项
- 题干不能重复
- 正确选项文本不能在同一批里反复复用
- 不要输出占位文案，比如“干扰项1”“与材料不一致的表述1”
- `evidence_quote` 不能为空

## 与前端兼容要求

1. 前端现在默认一次性生成，所以请把 `/questions/generate` 做成完整返回。
2. `/generation-jobs/{jobId}`、`/generation-jobs/{jobId}/batches/{batchIndex}` 可以先保留兼容，但正常情况下不应再依赖它们补题。
3. 如果短期内你们还没删旧批次逻辑，也请保证当 `/questions/generate` 已经返回整套题时，`generationJob.status` 为 `completed`，不要再让前端误以为还有后续批次。

## 落地优先级

1. 先保证 `/questions/generate` 一次性返回全量题目
2. 再保证 schema 和字段完全对齐
3. 最后再考虑是否下线旧 batch 接口
