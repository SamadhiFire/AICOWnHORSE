# 项目信息文档

整理时间：2026-03-11

## 1. 项目概述

本项目是一个基于 `uni-app + Vue 3 + TypeScript` 的学习刷题小程序，项目主体位于 `学习软件/`。

当前业务主线：

1. 用户在首页输入学习材料并设置出题参数。
2. 前端调用大模型生成题目，保存到本地题库与练习会话。
3. 用户在练习页答题，系统记录错题、掌握状态和练习进度。
4. 用户在题集页按分类和标签筛选复习。
5. 用户在“我的”页维护账号资料、LLM 配置和个性化标签。

## 2. 当前技术架构

当前项目已经切换为“前端 + 独立 REST 后端”架构，不再使用微信云开发云函数、云数据库或 `cloudfunctions/` 目录下的同步逻辑。

- 前端目录：`学习软件/`
- 后端基础地址：`https://eogoylqiqymp.sealosbja.site/api/v1`
- 鉴权方式：`Authorization: Bearer <token>`
- 登录方式：游客登录 `POST /auth/guest`
- 资料同步方式：通过 REST 接口同步，不再依赖微信云端对接

接口设计与联调规范以以下两份文档为准：

- `学习文档/前端联调对接指南.md`
- `学习文档/后端接口文档.md`

## 3. 前端页面

当前页面共 4 个：

| 路由 | 文件 | 说明 |
| --- | --- | --- |
| `pages/index/index` | `src/pages/index/index.vue` | 首页，负责材料输入、出题参数设置、开始生成 |
| `pages/practice/index` | `src/pages/practice/index.vue` | 练习页，负责答题、判题、续批生成 |
| `pages/collection/index` | `src/pages/collection/index.vue` | 题集页，负责查看题库、错题和掌握状态 |
| `pages/profile/index` | `src/pages/profile/index.vue` | 我的页，负责账号、头像、LLM 配置和标签 |

## 4. 当前核心前端模块

| 文件 | 作用 |
| --- | --- |
| `src/utils/backend-api.ts` | 统一 REST 请求与上传封装 |
| `src/utils/account.ts` | 游客登录、资料获取/更新、头像上传、鉴权状态缓存 |
| `src/utils/backend-user.ts` | 账号能力的页面级封装 |
| `src/utils/backend-sync.ts` | 标签、题库、练习会话、生成任务等后端同步封装 |
| `src/utils/llm.ts` | LLM 配置、本地缓存、Key 校验、模型调用 |
| `src/utils/question-bank.ts` | 本地题库读写与后端同步 |
| `src/utils/practice-session.ts` | 当前练习会话读写与后端同步 |
| `src/utils/user-tags.ts` | 用户标签读写、生成、历史题目重打标 |
| `src/services/practice-generation-service.ts` | 出题生成服务编排 |

## 5. 已接入后端能力

当前前端已经按文档完成以下接口对接：

- 账号：游客登录、获取当前用户、更新昵称头像、头像上传
- LLM：获取配置、保存配置、拉取可选模型供应商、校验 API Key
- 标签：获取标签、保存标签、生成标签、历史题目重打标
- 题库：全量同步、按条件查询、批量删除、答题提交、掌握状态更新、标签更新
- 练习会话：获取当前会话、保存当前会话、清空当前会话
- 生成任务：创建生成任务、拉取任务状态、触发后续批次、取消任务
